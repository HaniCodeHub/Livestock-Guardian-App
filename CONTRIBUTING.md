# Contributing to Livestock Guardian

Thank you for your interest in contributing to the Livestock Guardian project! We welcome contributions of all kinds.

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a feature branch (`git checkout -b feature/your-feature`)
4. Make your changes
5. Commit your changes (`git commit -m 'Add your feature'`)
6. Push to your branch (`git push origin feature/your-feature`)
7. Open a Pull Request

## Code Style

- Follow Android/Kotlin conventions for mobile code
- Follow PEP 8 for Python backend code
- Use meaningful variable names
- Add comments for complex logic

## Testing

Before submitting a PR:
- Ensure all tests pass
- Add tests for new features
- Test on multiple Android versions if possible

## Reporting Issues

When reporting bugs, please include:
- Steps to reproduce
- Expected behavior
- Actual behavior
- Device and OS information
- Screenshots/logs if applicable

## Feature Requests

We'd love to hear your ideas! Please create an issue describing:
- The feature you'd like
- Why you think it's valuable
- Potential implementation approach

## Code of Conduct

Be respectful and professional in all interactions. We are committed to providing a welcoming and inclusive environment.

Thank you for contributing!
